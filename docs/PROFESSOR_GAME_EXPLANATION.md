# 教授向け読み上げ説明

オープンキャンパスで使う評価用ミニゲームは、短いお手本音源を聞いて、4つの候補音源を「近いと思う順」に並べてもらうものです。

1ラウンドでは、最初にお手本の約2秒の音を聞きます。その後、AからDまでの4つの候補を聞き、ドラッグアンドドロップまたは上下ボタンで、近い順に並べ替えて「この順番で決定」を押します。1人あたり4ラウンドで、だいたい2分以内に終わる想定です。

画面上には特徴量名や研究上の狙いは出しません。参加者には、音を聞いて直感的に近い順に並べるゲームとして遊んでもらいます。氏名、学校名、連絡先、年代は取らず、参加コードだけ任意入力にしています。

内部的には、既に計算済みの特徴量分析を使い、高重複の特徴量は候補から外しています。また、同じラウンド内で似た意味の特徴量ばかりにならないよう、候補特徴量同士の相関が強すぎない組み合わせにしています。今回のラウンドでは、候補特徴量同士の最大相関が0.30未満になるようにしています。

ログには、回答順位、回答時間、各音源の再生回数、候補ID、対象特徴量、特徴量の重複度、候補音源の区間情報をサーバー側に保存します。途中でやめた場合でも、完了したラウンドは後から集計できます。

この設計の目的は、少人数でも「どの特徴量が人間の近さ判断に乗りやすいか」を広く拾えるようにすることです。特に、重複度の低い特徴量を優先して出しつつ、同じ意味の特徴量に偏ったデータにならないようにしています。
